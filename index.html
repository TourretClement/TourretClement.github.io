SAE 24 : Projet intégratif

Contexte : Ce projet a été réalisé en fin de première année.

Objectif : L'objectif de ce projet était de réaliser un réseau pour une petite entreprise.

Travail réalisé : Pour mener à bien ce projet, nous étions organisés en groupes de quatre personnes et avions pour contrainte de le réaliser dans un délai de deux semaines. Nous disposions de 4 ordinateurs, deux agissant en tant que clients et les deux autres en tant que serveurs, l'un sous Windows et l'autre sous Linux. De plus, nous avions à notre disposition un switch et un routeur. Notre première tâche consistait à permettre la communication entre les machines au sein du réseau, puis à les faire sortir du réseau. Ensuite, nous avions plusieurs tâches à accomplir pour améliorer la qualité de notre réseau, telles que la configuration de softphones entre les ordinateurs clients pour permettre une communication vocale. De plus, nous devions configurer un serveur web, téléphonique et DHCP. En outre, il était nécessaire de mettre en place un moyen de contrôler les ordinateurs à distance.

Résultat : Pour mener à bien ce projet, nous devions faire preuve de rigueur et travailler efficacement en équipe. L'enseignant était satisfait de notre travail.

SAE 22 : Mesurer et caractériser un signal

Contexte : Ce projet se réalise en partiel autonomie à l’université durant plusieurs travaux pratiques et dirigés.

Objectif : L'objectif de ce projet est de nous faire découvrir la programmation Matlab.

Travail réalisé : Dans une première partie, je réalise une autoformation pour apprendre comment fonctionne le langage Matlab. Ensuite, je génère un signal sinusoïdal sous Matlab. Par la suite, je le multiplie par un autre signal puis j'affiche le résultat avec la fonction 'plot'. Ensuite, j'ai à ma disposition un script qui permet de réaliser le spectre d'un signal, je l'applique donc à mon signal pour obtenir son spectre. Dans la deuxième partie du projet, je dois réaliser deux calculs d'intégrale de manière différentes. La première, la plus simple, est la méthode des rectangles. Je réalise les calculs théoriques puis je les applique sur Matlab pour obtenir le même résultat. La deuxième méthode est celle des trapèzes, je réalise donc mes calculs théoriques puis je les implémente dans Matlab. La troisième partie du projet consiste à calculer la valeur moyenne et efficace d'un signal sinusoïdal et carré. J'utilise donc la même méthode, je réalise mes calculs théoriques puis je les implémente dans mon code Matlab. La quatrième et dernière partie consiste à récupérer des données d'un travaux pratiques réalisé en télécommunication pour les implémenter dans Matlab. Je vais par la suite faire un lissage de ces données puis déterminer la bande occupée à -20 dB, je calcule la puissance du canal puis j'affiche le résultat.

SAE 23 : Mettre en place une solution informatique pour l'entreprise

Contexte : Ce projet se réalise en autonomie à l’université durant plusieurs travaux pratiques et dirigés.

Objectif : Mettre en place un outil de support pour le service informatique de l’université. Créer un site web de gestion de tickets de support. Le site devrait contenir des informations disponibles pour tous les utilisateurs du système. Un utilisateur peut créer un ticket de support, suivre l’avancement de ses tickets de support, et supprimer un ticket de support via une base de données.

Travail réalisé : Nous disposons de deux dossiers, un frontend et un backend. Le frontend, qui nous a été fourni, correspond à la structure de la page web, donc aucune interaction n’est possible avec celui-ci. Pour ce faire, nous devons coder dans le dossier backend, c’est-à-dire que les informations qui seront exécutées dans ce dossier ne seront pas visibles par l’utilisateur du site. Dans un premier temps, nous devons comprendre le code donné pour pouvoir le structurer dans notre dossier backend. Une fois la base du dossier backend réalisée, nous devons ensuite créer une base de données sous le logiciel MongoDB. Pour ce faire, nous avons besoin, sous Windows, de MongoDB Atlas, qui est un superviseur de base de données, et de MongoDB Compass, qui nous permettra d’accéder à notre base de données.

Résultat : Le travail que j'ai réalisé est fonctionnel.
